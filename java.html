[문제] 다음 Java 프로그램의 실행 결과를 쓰시오.

public class Main {
    public static void main(String[] args) {
        int[][] a = {{45, 50, 75}, {89}};
        System.out.println(a[0].length);
        System.out.println(a[1].length);
        System.out.println(a[0][0]);
        System.out.println(a[0][1]);
        System.out.println(a[1][0]);
    }
}
정답 보기
3
1
45
50
89
해설 보기

Java는 가변 배열(Jagged Array)을 지원합니다.

  1. a[0]은 {45,50,75}로 길이 3
  2. a[1]은 {89}로 길이 1
  3. 각 원소 접근은 표준 2차원 인덱싱

C와 달리 Java의 2차원 배열은 각 행의 길이가 다를 수 있습니다.

[문제 2] 다음 Java 프로그램의 실행 결과를 쓰시오.

class Parent {
    int x = 10;
    void display() { System.out.println("Parent: " + x); }
}
class Child extends Parent {
    int x = 20;
    void display() { System.out.println("Child: " + (x + super.x)); }
}
public class Main {
    public static void main(String[] args) {
        Child c = new Child();
        c.display();
    }
}
정답 보기
Child: 30
해설 보기
  1. Child의 x = 20, super.x = Parent의 x = 10
  2. x + super.x = 20 + 10 = 30

super 키워드로 부모 클래스의 멤버에 접근할 수 있습니다. 필드는 오버라이드되지 않고 하이딩(hiding)됩니다.

2021년 3회

[문제] 다음 Java 프로그램의 실행 결과를 쓰시오.

public class Main {
    static int count = 0;
    private static Main instance;
    private Main(){ count++; }
    public static Main getInstance(){
        if(instance == null) instance = new Main();
        return instance;
    }
    public static void main(String[] args){
        Main a = Main.getInstance();
        Main b = Main.getInstance();
        Main c = Main.getInstance();
        System.out.println(count);
    }
}
정답 보기
1
해설 보기

이것은 싱글톤(Singleton) 패턴입니다.

  1. 첫 번째 getInstance() 호출: instance가 null → 새 객체 생성 → count=1
  2. 두 번째, 세 번째 호출: instance가 이미 존재 → 기존 객체 반환 → 생성자 호출 안 됨

싱글톤은 인스턴스를 하나만 만들도록 보장하는 디자인 패턴입니다. 시험에서 자주 출제됩니다.

정답 보기

해설 보기

Java에서 배열은 참조 타입입니다. b = a는 배열을 복사하는 것이 아니라 같은 배열 객체를 가리키게 합니다.

따라서 b[1] = 2000을 하면 a[1]도 2000이 됩니다. 독립적인 복사본을 원하면 Arrays.copyOf()나 clone()을 사용해야 합니다.

2022년 3회

[문제] 다음 Java 프로그램의 실행 결과를 쓰시오.

public class Main {
    public static void main(String[] args){
        int[] score = {85, 65, 90, 75, 95};
        int[] rank = {1, 1, 1, 1, 1};
        for(int i = 0; i < 5; i++){
            for(int j = 0; j < 5; j++){
                if(score[i] < score[j]) rank[i]++;
            }
        }
        for(int i = 0; i < 5; i++)
            System.out.print(rank[i]);
        System.out.println();
    }
}
정답 보기
35241
해설 보기

이것은 순위(Rank) 알고리즘입니다. 모든 쌍을 비교하여, 나보다 점수가 높은 사람 수만큼 순위를 증가시킵니다.

  1. 85점: 90, 95보다 작음 → rank = 1 + 2 = 3
  2. 65점: 85, 90, 75, 95보다 작음 → rank = 1 + 4 = 5
  3. 90점: 95보다 작음 → rank = 1 + 1 = 2
  4. 75점: 85, 90, 95보다 작음 → rank = 1 + 3 = 4
  5. 95점: 자기보다 큰 점수 없음 → rank = 1

결과: 3, 5, 2, 4, 1 → 이어서 출력하면 35241

핵심: 초기 rank가 모두 1이고, 자기보다 높은 점수가 있을 때마다 1씩 증가시킵니다. 점수가 높을수록 순위 숫자가 작습니다(1등이 최고).

public class Main {
    public static void main(String[] args){
        String s1 = new String("java");
        String s2 = new String("java");
        String s3 = s1;
        System.out.println(s1 == s2);
        System.out.println(s1 == s3);
        System.out.println(s1.equals(s2));

정답 보기

해설 보기

2023년 3회

[문제] 다음 Java 프로그램의 실행 결과를 쓰시오. (다형성)

class A {
    void show(){ System.out.print("A"); }
}
class B extends A {
    void show(){ System.out.print("B"); }
}
class C extends A {
    void show(){ System.out.print("C"); }
}
class D extends B {
    void show(){ System.out.print("D"); }
}
public class Main {
    public static void main(String[] args){
        A a1 = new D();
        A a2 = new C();
        B b1 = new D();
        a1.show();
        a2.show();
        b1.show();
        a1.show();
    }
}
정답 보기
DCDD
해설 보기

다형성(Polymorphism): 실제 객체 타입의 메서드가 호출됩니다.

  1. a1 = new D() → 실제 타입 D → show()는 "D"
  2. a2 = new C() → 실제 타입 C → show()는 "C"
  3. b1 = new D() → 실제 타입 D → show()는 "D"
  4. a1.show() → 여전히 D

변수의 선언 타입(A, B)이 아니라 실제 생성된 객체의 타입(D, C)에 따라 오버라이드된 메서드가 실행됩니다. 이를 동적 바인딩이라 합니다.

[문제] 다음 Java 프로그램의 실행 결과를 쓰시오.

interface Calc {
    int sum(int from, int to);
}
class OddSum implements Calc {
    public int sum(int from, int to){
        int s = 0;
        for(int i = from; i <= to; i++)
            if(i % 2 != 0) s += i;
        return s;
    }
}
class EvenSum implements Calc {
    public int sum(int from, int to){
        int s = 0;
        for(int i = from; i <= to; i++)
            if(i % 2 == 0) s += i;
        return s;
    }
}
public class Main {
    public static void main(String[] args){
        Calc c1 = new OddSum();
        Calc c2 = new EvenSum();
        System.out.println(c1.sum(1, 10));
        System.out.println(c2.sum(1, 10));
    }
}
정답 보기
25
30
해설 보기
  1. OddSum: 1+3+5+7+9 = 25
  2. EvenSum: 2+4+6+8+10 = 30

인터페이스: Calc는 sum 메서드를 정의하고, 각 클래스가 구현합니다. implements 키워드를 사용합니다. 인터페이스 변수로 다형적 호출이 가능합니다.

[문제] 다음 Java 프로그램의 실행 결과를 쓰시오.

public class Main {
    public static void main(String[] args){
        int[] arr = {3, 1, 2};
        int temp = arr[0];
        arr[0] = arr[2];
        arr[2] = temp;
        for(int v : arr)
            System.out.print(v);
        System.out.println();
    }
}
정답 보기
213
해설 보기
  1. 초기: arr = {3, 1, 2}
  2. temp = arr[0] → temp = 3
  3. arr[0] = arr[2] → arr[0] = 2
  4. arr[2] = temp → arr[2] = 3
  5. 교환 후: arr = {2, 1, 3}
  6. 향상된 for문(for-each)으로 공백 없이 출력 → 213

배열의 첫 번째 원소(3)와 마지막 원소(2)가 교환됩니다. for(int v : arr)는 배열의 모든 원소를 순회하는 향상된 for문입니다.

public class Main {

Error
Done
해설 보기

try-catch-finally: 예외 처리의 기본 구조. finally는 리소스 정리 등에 사용됩니다.
